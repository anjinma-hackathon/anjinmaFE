# project: 교육약자를 위한 실시간 번역 서비스

# **PRD v4 — 실시간 번역 기반 교육약자 지원 서비스**

### *(교수/학생 페이지 분리 + 코드 기반 접속 + 로컬 저장 기능 완전 반영)*

---

# 1. 프로젝트 개요

목표:

경청이 어려운 학생들을 위해 **강의 음성을 자막·번역으로 실시간 제공**하는 서비스.

대상:

- 유학생 / 다문화 학생
- 청각 장애 학생
- ADHD·발달장애 학생
- 온라인 강의나 큰 강의실에서 소리가 잘 안 들리는 학생

---

# 2. 문제 정의

- 음성 중심의 강의는 특정 학생들에게 큰 학습 격차를 만듦
- 한국어가 약하거나 청각 정보 처리가 어려우면 강의 내용을 놓치기 쉬움
- 학교마다 실시간 자막 서비스가 없어 접근성 부족

---

# 3. 주요 사용자

- 교환학생/유학생
- 청각 장애 학생
- ADHD/발달장애 학생
- 교수자

---

# 4. 사용자 스토리

### 유학생

“수업 자체는 따라가는데 한국어 듣기는 너무 힘들어. 실시간 번역 자막이 필요해.”

### 청각 장애 학생

“교수가 말하는 내용을 글자로 볼 수 있어야 수업에 참여할 수 있어.”

### ADHD 학생

“중간에 집중이 끊겨도 자막이 있으면 놓친 내용을 따라갈 수 있어.”

### 교수자

“수업 시작할 때 방을 생성하고, 이미 만들었던 방에 다시 들어가서 수업을 이어가고 싶다.”

# 5. 핵심 기능

---

## ⭐ **5.1 교수 페이지 기능 (Teacher Page)**

교수 페이지는 **두 개의 기능만 존재**한다.

---

### **① 방 만들기 (Create Room)**

교수가 입력:

- 교수 이름
- 과목명

백엔드가 생성:

- **roomId**
- **교수코드 (teacherCode)**
- **학생코드 (studentCode)**

구성 예:

```
TeacherCode: ABC123
StudentCode: H97XQ2

```

사용:

- 교수코드 = 향후 다시 방에 들어올 때 사용
- 학생코드 = 학생에게 공유하여 수업 방 입장

---

### **② 교수코드로 기존 방 들어가기**

교수 페이지에서 입력 폼 제공:

```
[교수코드 입력] → [입장하기]

```

백엔드 검증:

- teacherCode → roomId 매핑 확인
- 방 존재하면 교수용 실시간 STT 페이지로 이동
- 없다면 에러 메시지

---

---

## ⭐ **5.2 학생 페이지 기능 (Student Page)**

학생 페이지는 좌우로 완전히 나뉨.

---

### **① 기존 방 목록(LocalStorage 저장)**

학생이 **한 번이라도** 특정 수업(studentCode)으로 입장했다면:

localStorage 예:

```json
[
  { "roomId": "1a2b3", "studentCode": "AJ2921", "professor": "김철수", "subject": "경영학원론" },
  { "roomId": "1c2d9", "studentCode": "BT7129", "professor": "박지현", "subject": "경제학" }
]

```

UI:

```
[📚 내 수업 목록]
- 경영학원론 (김철수 교수)
- 경제학 (박지현 교수)

```

목록 클릭 → 즉시 해당 방으로 입장

(코드 재입력 불필요)

---

### **② 학생코드로 새로운 방 들어가기**

오른쪽 패널:

```
[학생코드 입력] → [입장하기]

```

동작:

- server에 studentCode 전달 → roomId 검증
- 성공 시 해당 방에 입장
- localStorage에 해당 방 자동 저장
- 다음 방문부터 “기존 방 목록”에서 자동 표시

---

---

## ⭐ **5.3 실시간 강의 기능**

### 교수 화면

- STT(Web Speech API)로 음성 → 텍스트 변환
- 텍스트를 서버로 전송
- 서버가 번역
- Socket.io로 학생들에게 broadcast

### 학생 화면

- 실시간 자막 받기
- 언어 선택 (한국어/영어/중국어/베트남어/일본어)
- 글자 크기/배경색/테마 조정
- 스크롤 모드 자동/수동 선택

# 7. 기능 요구사항 (수정반영)

### FR-1

교수 페이지: 방 생성 + 코드 2개 발급

### FR-2

교수 페이지: 교수코드로 기존 방 검증 후 입장

### FR-3

학생 페이지: 학생코드로 신규 방 입장 가능

### FR-4

학생 페이지: 기존 방 목록을 localStorage로 영구 저장

### FR-5

실시간 STT → 번역 → 다수 학생에게 송출

### FR-6

PDF 교재 번역(선택)
